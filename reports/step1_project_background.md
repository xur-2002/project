# Step 1 项目背景定义

## 项目中文标题
内容平台核心指标监控、异常归因与 A/B Test 评估系统

## Project English Title
Content Platform KPI Monitoring, Anomaly Attribution, and A/B Test Evaluation System

## 业务背景
某本地生活内容平台以短视频与图文双内容形态承接推荐流量，近两个月在商业化投放和内容供给扩张后出现三个管理痛点：

1. 总曝光增长较快，但点击率与完播率出现阶段性回落，增长质量不稳定。
2. 某些高互动内容拉升消费时，举报率同步上升，平台生态风险增加。
3. 推荐策略改版频繁，团队缺少统一的实验评估框架，难以在增长指标和风险指标之间做上线决策。

业务方需要一套可复用的分析闭环，支持从日常监控到异常归因，再到策略实验评估的完整链路。

## 业务目标
1. 建立日级核心指标监控体系，覆盖流量、消费、互动、转化、风险。
2. 快速识别指标异常并定位主驱动维度，降低问题排查时间。
3. 形成 A/B Test 标准评估模板，平衡增长收益和生态护栏。
4. 沉淀可视化展示方案，提升运营、策略、内容治理协同效率。

## 分析目标
1. 设计可落地指标体系，明确结果指标、过程指标、风险指标、实验指标。
2. 搭建异常检测流程，采用移动均值偏离与环比突变结合的规则化方法。
3. 对异常进行分层归因，识别内容类目、流量来源、用户类型的贡献差异。
4. 对推荐卡片改版实验进行多指标评估，输出是否推广的业务建议。

## 岗位价值映射
1. 数据分析能力：将模糊业务问题拆解为可量化指标和可验证假设。
2. 商业分析能力：把增长、风险、转化放在同一决策框架下衡量。
3. 策略分析能力：通过归因分析定位主驱动问题并提出动作建议。
4. 数据产品意识：通过 dashboard 结构化输出可执行结论，支持跨团队沟通。

## 合理假设说明
1. 数据为日级聚合明细，按日期-类目-流量来源-用户类型-实验组别统计。
2. 转化统一定义为商业化目标行为完成数，如到店券领取或下单。
3. 举报以曝光为分母计算举报率，作为生态护栏指标。
4. A/B 实验在同一时间窗内稳定分流，样本满足基础可比条件。

## 本步产出
- 完成真实互联网内容平台业务场景设定。
- 明确业务目标、分析目标与岗位能力映射。
- 为后续数据结构与分析代码提供统一口径。
