# 内容平台核心指标监控、异常归因与 A/B Test 评估系统

## 1. 项目背景
本项目模拟本地生活内容平台在增长与生态治理并行阶段的典型问题：曝光增长但质量波动、举报风险抬升、策略改版评估标准不统一。目标是搭建从监控到归因再到实验决策的业务分析闭环。

## 2. 指标体系
- 结果指标：Exposure、CTR、CVR、Revenue
- 过程指标：Complete Rate、Interaction Rate
- 风险指标：Report Rate、风险曝光占比
- 实验指标分层：核心收益指标（CTR/CVR/Revenue）+ 护栏指标（Report Rate/Complete Rate）

## 3. 异常监控结果
- 使用 7 日移动基线 + 标准差阈值 + 偏离比例规则识别异常。
- 全局与关键分层联合监控，共识别 24 个异常点。
- 关键异常集中在三类场景：CTR 下滑、举报率抬升、CVR 下滑。

## 4. 归因分析（3 个 Case）
### Case 1：CTR 下滑（2026-01-20 至 2026-01-28）
- 整体变化：-0.003127
- 主驱动：entertainment 类目 + recommendation 流量
- 结论：推荐流量中的娱乐类供给质量下行是主因，新用户影响更明显。
- 建议：回溯异常时窗新增供给质量，临时降低低质素材分发权重。

### Case 2：举报率上升（2026-02-05 至 2026-02-12）
- 整体变化：+0.000242
- 主驱动：food 类目 + search 流量，晚间时段贡献更高
- 结论：搜索召回内容风险上升，生态压力集中在美食相关内容。
- 建议：加强搜索召回过滤与高风险时段审核阈值。

### Case 3：转化率下滑（2026-02-15 至 2026-02-22）
- 整体变化：-0.001513
- 主驱动：follow 流量 + new_user 人群
- 结论：新用户点击后转化链路效率下降。
- 建议：优化新用户转化路径文案与权益呈现，回收低意图流量。

## 5. A/B Test 评估
实验窗口：2026-02-01 起

| 指标 | A组 | B组 | Lift% | 显著性 |
|---|---:|---:|---:|---|
| CTR | 0.1018 | 0.1099 | +7.94% | 显著 |
| Conversion Rate | 0.0706 | 0.0751 | +6.41% | 显著 |
| Complete Rate | 0.5107 | 0.4905 | -3.96% | 显著 |
| Report Rate | 0.0019 | 0.0023 | +18.47% | 显著 |
| Revenue | 6,671,770 | 7,637,100 | +14.47% | 业务提升 |

结论：B 组在增长和收入上显著更优，但护栏指标恶化，建议灰度放量并叠加风险治理，不建议直接全量。

## 7. 结论与建议
1. 平台波动主要由少量关键分层驱动，不是全局均匀变化。
2. 监控体系需要从全局指标升级为全局+分层联合告警。
3. 实验评估应坚持收益与护栏并重，避免单指标驱动上线。
4. 建议沉淀常态化分析模板，提高运营与策略响应效率。
